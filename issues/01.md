### Problem
There is no single artifact describing "what periods/cities exist and where their files
live." The backend encodes the period in filenames (`assess_2026_clean.csv`,
`neighbourhoods_2026_recovered.geojson`); the frontend re-states the same facts by hand
(`dataSources.js` `YEARS = [2026]`, `ReportCard.jsx` hardcoded URL, the "2026 / 365,406
properties" copy). The two sides can silently disagree, and adding a year means editing
both.

### Fix
The backend's final pipeline step writes `website/public/data/manifest.json`. The frontend
reads it and derives everything period-specific. This is the one artifact that makes the
2027-button scenario real: backend writes a new manifest entry, every selector / map /
table / headline updates with no frontend edit.

### Proposed schema (v1 — versioned per §4.4)
```json
{
  "manifest_version": 1,
  "generated_at": "2026-05-29T00:00:00Z",
  "sections": {
    "property-assessment": {
      "default_city": "Edmonton",
      "default_year": 2026,
      "datasets": [
        {
          "city": "Edmonton",
          "year": 2026,
          "geojson": "/data/property-assessment/neighbourhoods_2026_recovered.geojson",
          "csv": "/data/property-assessment/neighbourhood_aggregates_2026.csv",
          "n_properties": 365406,
          "n_polygons": 402,
          "refreshed_at": "2026-05-19",
          "boundary_shapefile_vintage": "2023-01",
          "colour_scale": {
            "var": "median_assessvalue",
            "min": 103500, "q25": 352625, "median": 425125,
            "q75": 496188, "max": 1226000
          }
        }
      ]
    }
  }
}
```

### Acceptance
- [ ] Schema committed as a versioned contract under `pipeline/_shared/` (or `docs/contracts/`), date-stamped per §4.4.
- [ ] A backend step (new `10_write_manifest.R` or a `_shared` helper) emits it from the actual outputs of `06`/`07`/`08b` — not hand-written.
- [ ] `n_properties`, `colour_scale`, and file paths are read from the run, not literals.
- [ ] Manifest is append/merge-safe: re-running 2026 updates the 2026 entry; adding 2027 adds a sibling, neither clobbers the other.
- [ ] One worked example in the file showing a two-year (2026 + 2027) manifest.

### Notes
The colour-scale block is deliberately included so the frontend stops hardcoding
`PHASE1_STATUS §5` numbers in three places (`choroplethStyle.js`, the sidebar `<p>`, the
Legend). Whether the scale stays locked across years or is recomputed is a separate
decision (see #7) — the manifest just makes it data, not literal.
