### Problem
`05` and `06` read `data/raw/property_info_2026_20260519.csv` — a hand-saved, date-stamped
file. Unlike the assessment pull in `01` (which streams from the permanent `q7d6-ambg`
URL), Property Information has **no fetch path and a frozen filename**. A 2027 refresh has
no way to obtain or locate the 2027 version without manual edits.

### Fix
Fetch it the way `01` fetches assessments. `dkk9-cj3x` is a permanent dataset ID; pull the
current snapshot at refresh time and write to a `YEAR`-derived path (or archive folder per
the onboarding doc's local-snapshot requirement).

### Acceptance
- [ ] Property Information is fetched from `data.edmonton.ca/.../dkk9-cj3x/...` at refresh time (mirroring `01`).
- [ ] Output path derived from `YEAR`, no date literal in the filename consumed by `05`/`06`.
- [ ] Raw snapshot archived locally (gitignored) for reproducibility.
- [ ] `05`/`06` updated to the new path; row counts unchanged for 2026.
