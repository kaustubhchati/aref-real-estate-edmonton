### Problem
`ReportCard.jsx` hardcodes `CSV_URL = "/data/neighbourhood_aggregates_2026.csv"` — a bare
`2026` literal, no city/year resolution at all. It's a refresh dead-end and inconsistent
with the map's own `dataSources` pattern sitting one folder over.

### Fix
Resolve the CSV URL through the manifest (#1) / the shared resolver (#6), with the same
`(city, year)` controls the map uses. Reuse `OptionToggle`; don't build a parallel seam.

### Acceptance
- [ ] `ReportCard` resolves its CSV from the manifest, no `2026` literal.
- [ ] City/year controls present and consistent with the map page (shared resolver).
- [ ] Calgary / missing-year fall to the same empty-state treatment.
- [ ] 2026 table renders identically to today.
